# Roam Prints Studio — website

Static multi-page site with:

- a simplified `Home + Shop` structure
- a working contact form (Resend)
- optional file uploads (Vercel Blob)
- a live Instagram feed (Behold)

Built to deploy on Vercel from `github.com/roamcollective/roam-site`.

## What's in here

```text
roam-site/
├── index.html                  Home page
├── shop.html                   Main shop page
├── function-organizers.html    Redirects to shop
├── automotive.html             Redirects to shop
├── decor.html                  Redirects to shop
├── keychains.html              Redirects to shop
├── api/
│   ├── quote.js                Receives the custom-order form and emails it
│   └── blob-upload.js          Issues upload tokens for reference files
├── assets/
│   ├── banner.png
│   ├── favicon.png
│   ├── logo-badge.png
│   └── rp-mark.png
├── build.py                    Edit product data, copy, nav, and redirects here
├── package.json                Declares deps for the API functions
├── .env.example                Env var reference
└── .gitignore
```

All public `.html` files are generated by `build.py`. Edit `build.py`, then run
`python3 build.py` to rewrite the whole site.

## Shop setup

`SHOP_ITEMS` inside `build.py` controls the catalog.

- Change item names, prices, and summaries there.
- Add a Stripe Payment Link to an item's `buy_url` to switch that card from
  `Request this print` to `Buy now`.
- Leave `buy_url` blank if you want the product to route to the custom request
  form instead.

## Make it actually work — 3 things

### 1. Contact form (Resend) — required
1. Create a free account at https://resend.com and add an API key.
2. Verify a sending domain in Resend (e.g. your Roam Prints domain). Until then
   you can test with `onboarding@resend.dev` as the FROM address — but it only
   delivers to the email on your Resend account.
3. Set these env vars in Vercel (Project → Settings → Environment Variables):
   - `RESEND_API_KEY`
   - `QUOTE_TO_EMAIL` — where requests land
   - `QUOTE_FROM_EMAIL` — a verified-domain sender
4. Redeploy. Submissions now email you with every field filled in.

### 2. File uploads (Vercel Blob) — optional
In Vercel: **Storage → Create → Blob**, connect it to the project. That auto-adds
`BLOB_READ_WRITE_TOKEN`. Reference files (images, STL, 3MF, STEP up to 50 MB) then
upload directly and the link is included in the email. Without Blob, the form
still works — people just paste a reference link instead.

### 3. Instagram feed (Behold)
1. Make sure **@roamprintsstudio is a Creator or Business account** (free, 30 sec
   in Instagram settings). Personal accounts can no longer be embedded.
2. Create a free feed at https://behold.so, connect the account, make a **JSON**
   feed, copy its URL.
3. Open `build.py`, find `const BEHOLD_FEED_URL` inside `HOME_JS`, and paste
   your URL between the quotes. Then regenerate the site with `python3 build.py`.
   Until it's set, the section shows neutral placeholder tiles.

## Stripe note

If you use **Stripe Payment Links**, you do not need extra server-side Stripe
code for the first version of this site. Paste each payment link into the
matching product's `buy_url` inside `SHOP_ITEMS`, rebuild, and redeploy.

If you later want carts, quantities, or one checkout for multiple items, that is
when it makes sense to add Stripe Checkout sessions and server-side API logic.

## Deploy to Vercel via GitHub

**Push the code** to `github.com/roamcollective/roam-site`.

Preserving the repo's history (recommended):

```bash
git clone https://github.com/roamcollective/roam-site.git
# copy the contents of this folder into the cloned repo, overwriting old files
cd roam-site
git add -A
git commit -m "Rebuild Roam Prints site"
git push
```

Or, if you want to replace everything in the repo with a clean start:

```bash
cd roam-site            # this folder
git init
git add .
git commit -m "New Roam Prints site"
git branch -M main
git remote add origin https://github.com/roamcollective/roam-site.git
git push -u origin main --force   # --force overwrites existing repo contents
```

**In Vercel:** import the `roam-site` repo.

- Framework Preset: **Other**
- Root Directory: the folder containing `index.html`, `shop.html`, `api`, and `assets`
- Build Command: optional, but `python3 build.py` is safe if you want Vercel to
  regenerate the static files during deploy

Add the env vars above, then deploy. Point your domain at it in Project → Domains.

## Local preview

For a quick preview:

```bash
python3 -m http.server 3000
```

Then open `http://localhost:3000`.

The form and IG feed only fully work once deployed because they call `/api/...`
and an external feed, so local submit errors are expected unless you emulate the
serverless functions.
